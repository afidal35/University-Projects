Getting Started

Usage

Download

SSG is a static web site generator. this project was realized by a group of six students during the Software Engineering teaching Unit given at the University of Paris. Its use is quite simple:

Build your website

You can generate your web site from the target folder or by calling the ssg build command with the path of the target folder. The target folder must contain at least one folder with the name content and containing all markdown files.

From the current folder 

                
                    iren@mika:-$ cd path_to_website
                    iren@mika:-$ ssg build

Pass in parameters the target folder

The -i option allows you to specify a folder the path to another target folder other than the current folder. And the -o option allows you to specify the output folder 

                
                    iren@mika:-$ ssg build -i path_to_website -o output_path

Launch the server

An http server is integrated in order to easily visualize the results of the compilations. The server is started as follows:

Help Command

When ssg is invoked only, it displays the list of available commands.
And to get help on a command, just call ssg help command_name. For example ssg help build. 

                
                    iren@mika:-$ ssg
                    Usage  :

                    Usage: ssg [COMMAND]

                    Description  :

                    Static site generator : transform markdown files into html files to render a
                    website.


                    Commands  :

                      build    Convert markdown files (commonmark spec) into HTML files.
                               When specified with no options build from the current directory into
                                 the default output directory [_output].

                      serve    build project and launch it on a HTTP server for seeing the
                                 compilation's result
                      help     Displays help information about the specified command
                      version  Show current app version
                
            
                
                    iren@mika:-$ ssg help build
                    Usage:

                    Usage: ssg build [-rw] [--jobs=<jobs>] [-o=<output>] [<file1.html> |
                            [-i=<input>]]

                    Description  :

                    Convert markdown files (commonmark spec) into HTML files.
                    When specified with no options build from the current directory into the
                    default output directory [_output].


                    Parameters  :

                          <file1.html>          Convert one or multiple markdown files into html
                                                  files (10 maximum).

                    Options  :

                      -i, --input-dir=<input>  Build from the specified directory.
                          --jobs=<jobs>        Number max of job
                      -o, --output-dir=<output> Store built file(s) to the specified directory.
                      -r, --rebuild-all         Rebuild all the site.
                      -w, --watch               Watch file changes and recompile if needed.
